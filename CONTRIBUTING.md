# 🤝 Як долучитися до проєкту

Дякуємо за інтерес до нашого вебзастосунку **To-Do List**! Ми цінуємо будь-яку допомогу: від виправлення помилок до пропозицій нових функцій чи покращення документації.

## 📋 Порядок роботи з проєктом

1. **Форкніть репозиторій**
   Створіть власну копію репозиторію, натиснувши кнопку **Fork** у верхній частині сторінки GitHub.

2. **Клонуйте свій форк**
   Виконайте в терміналі команду:
   
       git clone https://github.com/weakbtw/Testing_PR2.git

3. **Створіть нову гілку**
   Використовуйте зрозумілі назви гілок (feature/, fix/, docs/):
   
       git checkout -b feature/назва-нової-функції

4. **Внесіть зміни та напишіть тести**
   Кожна нова функція повинна бути покрита модульними тестами у файлі `todo.test.js`.

5. **Запустіть тести**
   Переконайтеся, що всі тести проходять успішно:
   
       node --test todo.test.js

6. **Зробіть коміт та відправте зміни**
   Дотримуйтесь чіткого формату повідомлень:
   
       git add .
       git commit -m "feat: короткий опис змін"
       git push origin feature/назва-нової-функції

7. **Створіть Pull Request**
   На сторінці вашого форку натисніть **New Pull Request** і виберіть гілку для об'єднання.

## 📏 Правила стилю коду
* Використовуйте **camelCase** для назв функцій та змінних.
* Дотримуйтесь існуючої структури класів.
* Відступи — **2 пробіли**.
